COMP-2310 Chapter 2 - First-Order Logic Overview

Justin Bornais • October 4, 2023
education computer-science

First-order logic is essentially an extension of propositional logic. Thus, the same equivalences and inference rules applies in FOL. However, FOL is intended to expand on propositional logic by including functions. Here’s what I mean:

  • If I wanted to say “John goes to the movies”, “Sam goes to the movies”, “Alex goes to the movies”, I could use JJJ, SSS and AAA to represent John/Sam/Alex (respectively) going to the movies.
  • However, instead, what if I defined a function M(x)M(x)M(x) for any person xxx where M(x)=x  goes  to  the  moviesM(x) = x\;goes\;to\;the\;moviesM(x)=xgoestothemovies?
  • That way, I could have M(j)M(j)M(j), M(s)M(s)M(s), M(a)M(a)M(a) for John/Sam/Alex (respectively) going to the movies. These are called predicates.
  • What makes this even better is I can have an expression for “Everyone goes to the movies”: (∀x)M(x)(\forall x)M(x)(∀x)M(x) or “There exists someone who goes to the movies”: (∃x)M(x)(\exists x)M(x)(∃x)M(x).
  • The ∀\forall∀ and ∃\exists∃ operators are called quantifiers.

Note

The restriction ”α(x)\alpha(x)α(x) does not contain yyy” on FE1 and FE2 can be illustrated in the following way:

1. (∀x)(p(x)∧q(y))(\forall x)(p(x)\land q(y))\quad(∀x)(p(x)∧q(y)) from Γ\GammaΓ
2. (∀y)(p(y)∧q(y))(\forall y)(p(y)\land q(y))\quad(∀y)(p(y)∧q(y)) 1, FE1

Line 2 is invalid, since the expression contains a predicate that contains yyy. Now I quantified yyy in q(y)q(y)q(y), which is nonsensical.

Free Variables

In the definition of the above equivalences and inference rules, you saw the term free occurrence or free variable. A free variable is simply a variable that is not quantified. A bound variable is one that is quantified.

  • For instance, if I have the expression (∀x)p(x)∧y(\forall x)p(x)\land y(∀x)p(x)∧y, the quantifier (∀x)(\forall x)(∀x) quantifies xxx. Since yyy is not quantified, then yyy is a free variable.
  • However, if I had the expression (∀y)((∀x)p(x)∧y)(\forall y)((\forall x)p(x)\land y)(∀y)((∀x)p(x)∧y), then yyy is no longer free as it is quantified by the (∀y)(\forall y)(∀y) quantifier. Thus it is a bound variable.
    • Note: You are almost never going to see an expression like this in FOL. yyy will usually belong to a predicate.

Updated Definition of a Proof

Along with these, there is a refined definition of proofs:

A derivation or proof of a FOL-wff α\alphaα from a set of FOL-wffs Γ\GammaΓ is a sequence of FOL-wffs, α1,α2,…,αn\alpha_1,\alpha_2,\dots,\alpha_nα1​,α2​,…,αn​ such that:

  1. αn=α\alpha_n=\alphaαn​=α, and
  2. For each i,1≤i≤ni,1\leq i\leq ni,1≤i≤n,
    1. αi\alpha_iαi​ is a FOL-wff from Γ\GammaΓ, or
    2. αi\alpha_iαi​ is of the form (ω∨∼ω)(\omega\lor\sim\omega)(ω∨∼ω) called axiom, where ω\omegaω is a FOL-wff, or
    3. αi\alpha_iαi​ is a direct consequence of some of the preceding FOL-wffs by virtue of one of the inference rules or by substitution, and
  3. Each application of EI introduces a new constant which does not appear in α\alphaα, and
  4. No application of Gen using xxx is made to α(x)\alpha(x)α(x), if the variable xxx appears as a free variable in a hypothesis upon which α(x)\alpha(x)α(x) is deduced, and
  5. No application of Gen is made using a variable that is free in some (∃x)β(x)(\exists x)\beta(x)(∃x)β(x) to which EI has been previously applied.

The first two lines are the same as in chapter 1. However, lines 3, 4 and 5 are new. Notice these rules describe only EI and Gen. Here is what they all mean:

3. Each application of EI introduces a new constant which does not appear in α\alphaα

When using EI, you have to introduce a brand new constant. You can’t reuse a constant that already exists, because you’re assuming the same entity that satisfies some other expression also satisfies the current expression.

However, for UI, you can optionally introduce a new constant. That’s why you should use EI before UI.

Example

The following proof is invalid:

1. (∃x)Even(x)(\exists x)Even(x)\quad(∃x)Even(x) from Γ\GammaΓ
2. (∃x)Odd(x)(\exists x)Odd(x)\quad(∃x)Odd(x) from Γ\GammaΓ
3. Even(a)Even(a)\quadEven(a) 1, EI, aaa is a new constant
4. Odd(a)Odd(a)\quadOdd(a) 2, EI, aaa is a constant
5. Odd(a)∧Even(a)Odd(a)\land Even(a)\quadOdd(a)∧Even(a) 4, 3, I6
6. (∃x)(Odd(x)∧Even(x))(\exists x)(Odd(x)\land Even(x))\quad(∃x)(Odd(x)∧Even(x)) 5, EQ

Assume Odd(x)Odd(x)Odd(x) defines a predicate for ”xxx is odd” and Even(x)Even(x)Even(x) defines ”xxx is even”. By reusing the same constant, we are assuming the same constant satisfies both predicates, which is obviously nonsensical (how can a number be both odd and even?).

4. No application of Gen using xxx is made to α(x)\alpha(x)α(x), if the variable xxx appears as a free variable in a hypothesis upon which α(x)\alpha(x)α(x) is deduced

If I start with C(x)C(x)C(x) and then later say (∀x)C(x)(\forall x)C(x)(∀x)C(x). then I’m basically saying I started with some expression such as C(m)C(m)C(m) (i.e. Max likes chocolate) and am now concluding that every possible value for xxx satisfies C(x)C(x)C(x). I’m essentially saying “Max likes chocolate. Therefore, everybody likes chocolate.” This is nonsensical.

Effectively, if you start with a predicate that is not quantified, then you cannot perform Gen on it.

5 No application of Gen is made using a variable that is free in some (∃x)β(x)(\exists x)\beta(x)(∃x)β(x) to which EI has been previously applied

If you used EI for some expression, you can’t use Gen later. If you start with (∃x)C(x)(\exists x)C(x)(∃x)C(x) (“There exists someone who likes chocolate”), then use EI to get C(m)C(m)C(m) (“Max likes choocolate”), you can’t later use Gen to get (∀x)C(x)(\forall x)C(x)(∀x)C(x) (“Everyone likes chocolate”). That is nonsensical.
